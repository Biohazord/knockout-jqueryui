Extensibility

The library has a factory object that can generate bindings for any jQuery UI widget. It has a single method:

{% highlight js %}bindingFactory.create(config){% endhighlight %}

It creates the new binding based on the received configuration object. The configuration object has the following properties:

name
String. The widget's name.
options
Array of strings. Names of the widget's options. The factory creates a binding option for each of the widget's options. When any of their values change, the binding will set the widget's corresponding option to the new value.
events
Array of strings. Names of the widget's events. The factory creates a binding option for each of the widget's events. Their values have to be functions. The binding will bind them to the widget's events.
hasRefresh
Boolean, optional. Indicates whether the widget has a refresh() method. Most of the jQuery UI widgets has. When set to true, then the factory creates a binding option named 'refresh'. Changes in the value of this option will cause the binding to invoke the widget's refresh() method. This is useful for refreshing container-like widgets (tabs, accordion) when their items change. See the example on the tabs binding's documentation.
preInit
Function, optional. The binding will invoke this function before it initializes the widget.
postInit
Function, optional. The binding will invoke this function after it initializes the widget. This is used for implementing the bindings' non-standard options, e.g. the dialog binding's isOpen option.
